**Q：待开发的产品找不到合适的品类？**
→ 请联系平台对接人沟通新增品类。

**Q：登录时无法收到邮件验证码？**
→ 检查邮箱地址是否正确、验证邮件是否被过滤到垃圾箱。海外 ODM 环境避免使用国内域名邮箱（qq、163 等），尽量使用 Gmail 等海外域名。

**Q：在平台找不到合适的产品功能来匹配硬件的能力？**
→ 请联系平台对接人沟通新增自定义功能。

**Q：调试设备无法连接平台？**
→ 确认设备配网正确，检查五元组是否烧录正确，PID 与 deviceSecret 是否一致。

**Q：我应该配置哪些向导配置？**
→ 取决于产品联网方式，建议如下：1. **Wi-Fi 产品 **— 配置两种向导：蓝牙添加、WiFi 配网
蓝牙添加：第一优先级配网绑定
WiFi 配网：备选方案
**2****.**** ****4G 产品 **— 配置四种向导：蓝牙添加、扫设备机身码、设备扫码、ID 添加
蓝牙添加：第一优先级绑定
扫设备机身码：第二优先级
ID 添加 / 设备扫码：备选方案
**LAN 产品** — 配置三种向导：扫设备机身码、局域网扫描添加、ID 添加
扫设备机身码：第一优先级绑定
局域网扫描添加 / ID 添加：备选方案
**Q：为什么功能分组中并未设置的功能，也在 APP 配置页面中展示了？**
→ 功能分组仅控制 APP 配置页面中的功能设置部分，其他板块（产品基本信息、重启设备、分享设备、关于本机等）均为 APP 内置。
******Q：功能分组中配置的功能为什么无法在 APP 中控制？**
→ 平台当前仅支持 Bool 和 Enum 类型的属性功能在 APP 侧控制，其他功能仅展示设备最新值。平台将逐步开放。
******Q：什么是 STQC，为什么固件包上传时提醒必须通过 STQC 认证？**
→ 请联系平台对接人了解认证流程。
******Q：创建固件升级任务时，为什么无法选中某个固件版本？**
→ 请检查历史固件升级任务是否已关联该固件版本，系统严格控制每个固件版本仅可在一个产品使用升级一次。
******Q：我已经完成了固件升级任务的 OTA 验证，如何进行批量 OTA？**
→ 设备批量 OTA 功能尚未对开发者开放，请联系平台侧，告知已创建的固件升级任务由平台侧操作。
******Q：为什么我的设备在某些 APP 中搜索不到？**→ 请检查该设备所属的产品 PID 是否已关联到目标 APP。进入【产品开发】→【设备交互】→【关联应用】查看该 PID 的关联 APP 列表。

**Q：如何更改设备扫码后的默认下载 APP？**→ 在【关联应用】配置中修改「主应用」设置。注意需在产品「开发中」状态下修改，切换至「预发布」后生效。
******Q：已绑定到 APP 的设备，更改关联关系后会受影响吗？**→ 不会。已绑定的设备使用不受影响。但该设备解绑后，重新添加时将受新规则约束。
