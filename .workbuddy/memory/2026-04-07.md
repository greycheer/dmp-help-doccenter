## 表格排查 + 图片尺寸调整
- **表格未展示原因**：飞书文档中的表格实际是 block_type 30（多维表格/sheet），不是 block_type 22（普通表格/table）
  - 多维表格通过独立 token 引用，需飞书 bitable API 获取数据
  - 当前应用权限返回 `NOTEXIST`（91402），无法访问多维表格
  - 需要在飞书开放平台为应用授予 `bitable:app` 相关权限
- **图片尺寸调整**：所有图片高度限制为 `screenHeight / 3`，按比例缩放宽高；竖版图额外限宽 360px

## 新版表格支持（block_type 31）
- 用户将飞书多维表格改为普通表格后，发现 API 返回的是 block_type 31（新版表格容器）+ 32（单元格）
- block_type 31 结构：`table.property`（column_size, row_size, header_row）+ `table.cells`（cell ID 数组，行优先）
- block_type 32 是单元格，内容在其 children（type 2 text blocks）中
- 飞书 API `/blocks` 返回所有层级 blocks（含嵌套），需过滤 `parent_id === docToken` 只保留顶级
- pipeline 修改：
  - BT 映射新增 `31: 'table_new', 32: 'table_cell'`
  - `fetchNewTableAsMarkdown()`: 异步获取表格 children (type 32) → 每个 cell 的 children (type 2 text) → 渲染 Markdown 表格
  - `blocksToMarkdown`: block_type 31 时用 `block.table`（不是 `block.table_new`）获取属性
  - `main()`: getAllBlocks 后过滤 `parent_id === docToken`，只保留顶级 blocks
  - 同时支持 block_type 22（旧表格，cells 内联）和 31（新表格，cells 作为子 blocks）
  - 保留 bitable API 支持（block_type 30）作为后备
- 最终效果：Admin + Developer 两份文档的术语表格均正确渲染为 Markdown 表格（3列×9行）
