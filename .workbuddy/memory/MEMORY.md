## Architecture
- `pipeline.js` — 飞书文档同步脚本 v2.2（飞书API → 标题树 → 多级目录 + 文件）
  - `buildHeadingTree()`: 栈模式构建标题树，容忍跳级，过滤空标题
  - `writeTreeToDocs()`: 递归生成子目录 + `_category_.json` + 叶子 `.md`（async）
  - `syncImages()`: 图片增量下载维护
  - `blocksToMarkdown()` 增强: 下划线修正、裸URL图片、列表depth、callout→admonition、代码块纯文本化、移除<>转义、新版表格(type 31)
  - `fetchNewTableAsMarkdown()`: 异步获取 block_type 31 表格的 cell children 并渲染 Markdown 表格
  - `fetchBitableAsMarkdown()`: 异步获取 block_type 30 多维表格（需 bitable API 权限）
  - `stripHeadingNumber()`: 去除标题编号（`^\d+(\.\d+)*\.?\s+`），点号可选
  - `DOC_CONFIGS.sidebarLabel`: 控制侧边栏一级目录名（pipeline 不再覆盖手动改的 label）
  - 跳过 `docs/index.md` 生成（自定义 React 首页存在时）
  - `getAllBlocks()` 后过滤 `parent_id === docToken` 只保留顶级 blocks
- `docusaurus.config.js` — Docusaurus 配置（搜索、导航、主题、明暗切换）
  - `routeBasePath: '/docs'`（根路径 `/` 是自定义 React 首页）
  - `colorMode: { defaultMode: 'light', respectPrefersColorScheme: true }`
- `sidebars.js` — 使用 `autogenerated` 模式，根据目录结构和 frontmatter 自动生成侧边栏
- `src/pages/index.js` — React 自定义首页（Hero + Quick Start + Features + CTA）
- `src/css/custom.css` — 完整设计系统（~500行）：Inter 字体、Blue 主题、280px 侧边栏、玻璃态 navbar、暗黑主题、响应式、无障碍
- `docs/admin-manual/` — 管理员操作手册（侧边栏 label: "For Admin User"）
- `docs/developer-manual/` — 开发者操作手册（侧边栏 label: "For Developer"）
- `static/img/logo.svg` — SVG logo 图标（物联网主题：hub + 信号波纹 + 设备节点）
- `static/img/` — 图片资源（从飞书下载 / GitHub已有）
- `src/clientModules/imageLightbox.js` — 图片点击放大 lightbox + 竖版图片宽度限制
- **侧边栏目录名**：一级目录已缩短为 "For Admin User" / "For Developer"（category slug: `for-admin-user` / `for-developer`）
- **Navbar Logo**：SVG 图标 + 渐变艺术字（蓝→紫渐变），dark mode 自动适配
- Node.js 环境: `node-v20.19.0-win-x64/` 在项目目录（install_binary 有进程锁定问题）

## Feishu API
- APP_ID: cli_a944f3bde3b89bce
- block_type 是数字，不是字符串（2=text, 3=H1, 4=H2, 12=bullet, 27=image, 30=sheet/bitable, 31=table_new, 32=table_cell 等）
- text_run.content 包含文本，text_element_style 包含样式
- 图片下载需要 token，有频率限制
- **飞书表格 block 类型**：
  - type 22（旧普通表格）：cells 内联在 block.table 中（`cells['r-c']` 格式）
  - type 31（新普通表格）：`table.property`（行列数）+ `table.cells`（cell ID 数组，行优先），cell 内容通过子 blocks API 获取
  - type 30（多维表格/sheet）：`sheet.token` 引用，需 bitable API 获取（应用需授权）
- **`/blocks` API 返回所有层级 blocks**（含嵌套在表格中的 type 32 和其子 text blocks），需按 `parent_id` 过滤只保留顶级

## Important Lessons
- sidebars.js 的 doc ID 必须与文件路径精确匹配（去 .md 后缀），否则侧边栏条目不显示
- 最佳方案：用 `autogenerated` 模式 + frontmatter 的 `sidebar_position`，避免手动维护 ID
- 本地环境无 node，推送文件用 PowerShell `Invoke-RestMethod` + GitHub Contents API
- **Docusaurus MDX 对裸 `<` 处理并不总是安全**：`<*something` 会被当作 JSX 解析失败。pipeline 中用 `<(?![a-zA-Z\/\!?\#])` 精准转义
- deploy.yml 中 sync 步骤不 push 新文件回 repo，sync 仅在 CI 构建时临时生成
- PROTOTYPE 4.5g 的设计（"直接删除转义"）不完全正确，MDX 仍需转义部分 `<`
- **飞书嵌入多维表格(bitable)无法通过 API 读取**：block_type 30 的 sheet token 不被当前应用的 bitable API 识别（NOTEXIST 91402），需在多维表格本身授权给应用
- **飞书 API `/blocks` 返回含嵌套子 blocks**：表格 cell 内容(type 32) 和其 text children 会混入列表，必须按 parent_id 过滤
- **block_type 31 属性 key 是 `table` 不是 `table_new`**：BT 映射名称和实际 block 属性名可能不一致
- **Docusaurus autogenerated 模式自动去掉数字前缀**：目录名 `01-introduction` 的 slug 是 `introduction`，文件名 `04-quick-guide.md` 的 slug 是 `quick-guide`。链接路径中不能包含数字前缀
- **Docusaurus category 目录不自动生成 index.html**：`_category_.json` 设置 `generated-index` 只影响侧边栏显示，不创建独立页面。链接需指向 category 下的具体文档页面（如 `section-overview`）

## Update Workflow
1. 运行 `node pipeline.js` 或 `npm run sync` 从飞书同步最新内容（本地无 node，需在有 node 的环境执行）
2. 本地构建验证：`npm run build`
3. 推送到 GitHub（用 `push_to_github.js` 或手动 push）
4. GitHub Actions 自动部署
