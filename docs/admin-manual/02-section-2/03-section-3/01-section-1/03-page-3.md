2.2.1.3.1 **Thing Model**

A Thing Model describes device-level dynamic information under this category, such as alarm switch status and whether motion detection is enabled. A Thing Model can be either a mandatory Thing Model or an optional Thing Model, and can be flexibly adjusted via the "Required" toggle. When a product is created under this category, it can inherit this set of mandatory/optional Thing Models.

![](/img/admin-manual/en/w_0008.png)

2.2.1.3.2 Functional Grouping

Functional Grouping is used to manage the configurable items of a product on the APP and CMS sides, meeting the common configuration requirements when consumers use the product. When a product is created under this category, it can inherit the Functional Groupings under this category.![](/img/admin-manual/en/w_0009.png)
