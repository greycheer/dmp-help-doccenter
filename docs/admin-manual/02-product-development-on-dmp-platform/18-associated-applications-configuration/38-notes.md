Modifiable only in "In Development" status: The PID-App association can only be adjusted when the product status is "In Development". Once the product enters "Pre-Release", the association is locked and takes effect.
Already-bound devices unaffected: After the association takes effect, devices already bound to an app continue to function normally. Only future device binding for new devices is affected. If a previously bound device is unbound and re-added, the new rules will apply.
Single device binding limitation: A single device can be bound to only one app and one account at any given time.
Primary App auto-reassignment: If the association with the current Primary App is removed, the Primary App designation will be automatically reassigned to another associated app under the same product. If no other associated apps exist, the Primary App will be empty.
