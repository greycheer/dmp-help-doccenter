Category Management

DMP Platform Manual for Admin User · Last updated: Apr 7, 2026

The DMP management console provides a set of pre-built product categories based on business experience and market classification. When creating a product, you must select the correct product category. The system will preset the corresponding standard function library and configuration templates based on the category characteristics.

💡
Note: Product categories cannot be changed after product creation. Please select carefully.

Category Types

Categories are organized in a tree structure with two types:

TypeDescriptionCapabilities
Parent CategoryA non-leaf node in the category treeCan add sub-categories, but cannot add products
Leaf CategoryA terminal node in the category treeCan add products, but cannot create sub-categories

Navigation Path

[Product] → [Category Management] → [Backend Categories]

Figure: Category Management interface in the DMP console

Operations

Creating a Category

  1. Navigate to [Product] → [Category Management] → [Backend Categories]
  2. Click "Create Sub-item" or "Add Top-Level Category"
  3. Fill in the category name and other required information
  4. Specify whether the category is a leaf node
  5. Click Confirm to save

Editing a Category

Hover over the target category and click the edit icon. Modify the category name or leaf node status as needed.

⚠️
Warning: Changing a category from leaf to non-leaf (or vice versa) may impact existing products. Ensure no active products are bound to the category before making changes.

Deleting a Category

Categories with sub-categories or associated products cannot be deleted. Remove all children first, then proceed with deletion.

// Category structure example
Smart Home (Parent)
├── Lighting (Leaf)          ← Products can be added
│   ├── Smart Bulb
│   └── Smart Switch
├── Security (Leaf)
│   ├── Camera
│   └── Door Lock
└── Appliances (Parent)
    ├── Air Conditioner (Leaf)
    └── Refrigerator (Leaf)

Standard Functions by Category

Each leaf category comes with a predefined set of standard functions:

Function TypeDescriptionRequired?
AttributeReal-time device state (bool, number, string, enum, float)Varies
EventDevice-triggered notifications or alertsVaries
ActionExecutable commands sent to devicesVaries
✅
Tip: Standard functions defined by SDK developers (DP ID starting with "1") cannot be modified by other departments.
